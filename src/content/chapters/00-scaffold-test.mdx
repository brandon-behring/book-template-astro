# Scaffold test chapter

This file exists to exercise the chapter-rendering pipeline during Stage 0:

1. `src/content.config.ts` validates the frontmatter (Zod schema).
2. `src/pages/[...slug].astro` emits a static path for this entry.
3. `src/layouts/Chapter.astro` renders the ChapterHeader, ChapterTOC, and
   ChapterNav around the MDX body.
4. The route `/00-scaffold-test/` serves the result.

## What you should see

The chapter header above surfaces the frontmatter: part, chapter number,
last-verified date, volatility badge, and the tools-compared row. The
page heading (*h1*) was rendered from the MDX frontmatter `title` field.

## Testing the TOC

The TOC auto-hides when the chapter has fewer than three `h2`/`h3`
headings. This chapter has three, so it renders. Open it to jump around.
